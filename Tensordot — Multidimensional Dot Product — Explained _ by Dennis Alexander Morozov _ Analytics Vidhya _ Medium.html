Tensordot from Scalars to Multidimensional Tensors

The multidimensional operator, axes destroyer and dimensional transformer, tensordot has earned its rightful place in the coliseum of super useful multi-dimensional matrix operators. Tensordot is a powerful tool for tensors (multi dimensional matrices), but it can also be a bit elusive and seem slightly magical. The difficulty in gaining a robust intuition for tensordot is mostly due to the fact that if you need a tensordot operation, your mind is likely already dazed with keeping track of all the dimensions of a tensor. To get a full grasp of tensordots, it helps to start from simple matrices then slowly build an intuition all the way up to multidimensional operations.

Here are a few reasons to master tensordot:

  1. Tensordot is a super useful tool in tensor operations applicable in optimization, engineering, machine learning … or anywhere else you need to perform multidimensional operations.
  2. You can speed up you code by using tensordot instead of nested loops for multidimensional dot-products.
  3. It will help you build an intuition for multi-dimensional operations in general, and give you experience with tensors.
  4. Once you fully understand tensordot, it tends to stick. If you already have some experience with matrices, the the operation is not theoretically rigorous, it’s more of an visual/perceptual expansion of what you already know.
  5. We will look at numpy tensor dot specifically, however the intuition is applicable to many libraries that make use of tensors (tensorflow, pytorch…).

Before we dive into tensors, lets review a few essentials, then build from there to more complex tensordot operations. Although there is some math theory behind tensors, for our purpose, tensors are just matrices with more than 2 dimensions. In fact, we refer to all matrices as tensors, even the 2D matrices. Axes, and Axis Designations

A 2D matrix has two axes, the convention is to number each axis first along the rows then along columns, as shown below. This can be confusing because each instance of axis 0 is a column, while each instance along axis 1 is a row. The reason I use the word ‘along’ axis 0 or 1, is to demonstrate that we travel along the specific axis to extract an instance or a record of that axis. In tuple form we can say that this matrix is (2,3). The first number in the tuple represents the size of axis 0, and the second number represents the size of axis 1. In other words, a single instance of axis 0, is a row, and there are two rows in our matrix. A single instance of axis 1 is a column, and there are three columns in our matrix.

The same principle applies to more than 2 dimensions so we can extend our notation and visual representation as show below. Now our matrix is (2,3,depth, …N). Same as before, the length along axis 0 is 2, the length along axis 1 is 3, the length along axis 2 is depth… I stopped specifying actual lengths to abstract the understand into any dimension with Axis N.

2D Dot Product

Now let’s review a simple dot product for 2 matrices both in two dimensions. As you can see below, we take each row (each instance along axis 1) from X and each col (each instance along axis 0) from Y, multiply the row and col element wise, them sum the outputs together. Each element in the output matrix is composed of all possible combinations of rows from X with columns from Y.

Here is a sneak-peak for np.tensordot. Notice the axes notation, that is a critical part of understanding multidimensional tensordot operations. First nested tuple in the axes parameter refers to the axis along x, while the second nested tuple refers to axis along y.

x
array([[1, 2, 3],
       [2, 3, 4]])
y
array([[4, 5],
       [5, 6],
       [6, 7]])np.dot(x,y)same asnp.tensordot(x,y, axes = ((1),(0)))Out:
array([[32, 38],
       [47, 56]])

A simple dot product in 2D with np.dot(x,y) does the axis designation automatically for us, for multidimensional operations we need to specify along which axes we want the multiplication/summation to occur.

Tensordot with Vectors

Let’s take a few steps back from matrix dot product and start from scratch, tensordot with vectors. Tensordot with vectors is useful to build a strong intuition. It follows same patters as a matrix dot product, the only difference here is that we will look at dot product along axes specified by us.

First, lets create two vectors.

x = np.array([1,2,3])
y = np.array([4,5,6])

From numpy documentation, axes parameter accepts the following inputs:

np.tensordot(x,y,axes)

axes int or (2,) array_like

  • integer_like If an int N, sum over the last N axes of a and the first N axes of b in order. The sizes of the corresponding axes must match. When axes is integer_like, the sequence for evaluation will be: first the -Nth axis in a and 0th axis in b, and the -1th axis in a and Nth axis in b last
  • (2,) array_like Or, a list of axes to be summed over, first sequence applying to a, second to b. Both elements array_like must be of the same length

Personally, I didn’t find the explanation very helpful although it was a good starting point. To fully explore the inputs for the axes parameter, we will practice with both types of inputs: integer and nested tuples. Both are related, the integer input is a simplification of more common cases of dot-products, which can also be performed by specifying the axis along which to multiply/sum each with nested tuples.

For tensordot with axes = 0, take the last zero axis of x, each instance of x, and multiply/sum that with y. The wording seems awkward here since it’s not clear what the last zero axis of x is, but it will become more clear as we go on. I am trying to use same wording as numpy documentation in order to build the understanding. The last zero axis for x is just the scalar elements of x.

The dimensions change as follows (3)*(3) => 3x3 since there are a total of 9 unique combination of x with y. Each instance of x is a scalar, we only end up multiplying x[i] by y.

np.tensordot(x,y, axes = 0)
array([[ 4,  5,  6],
       [ 8, 10, 12],
       [12, 15, 18]])

The visual below shows how two vectors of length 3 transform into a 3x3 matrix.

When the axes parameter is 1, the dot product is along the full instance along the 0 axis for x, and 0 for y and perform the dot product (multiply then add). The strange wording would go as follows, take the last one axis of x, and perform the dot product with the last one axis of y.

x = np.array([1,2,3])
y = np.array([4,5,6])

The visual below shows the simple dot-prod between two vectors. This notation is commonplace in school because you can represent an equation with weights and variables (x+2y+3z). The notation can be combined to 2D and represent a system of equations.

We can’t explore further what np.tensordot(x,y, axes = 2) does when the axes parameter = 2 with vectors because vectors are one dimensional. We will either get an IndexError or a ValueError if we try to add axes to the tuple notation for axes. Let’s now go to a 2 dimensional tensor and keep exploring tensordot.

Tensordot with 2D Tensors

Remember that as far as we are concerned, a tensor is just a matrix with more than 2 dimensions. I will continue to refer to the matrices in the examples as tensors.

Lets revising numpy’s documentation for axes parameter and input type INT (N). This is where the strange wording come from.

Sum the last N axes of a, and the first N axes of b in order.
When axes is integer_like, the sequence for evaluation will be: first the -Nth axis in a and 0th axis in b, and the -1th axis in a and Nth axis in b last

If we represent the shape of a tensor with a tuple, the below image interprets the numpy definition, and can be extended to N axes.

Tensordot With Parameter Axes = 0

When N = 0, we don’t operate on an axis but on each scalar value of x. The operation can be worded the following way: perform the dot product for every instance of x with every instance of y.

Here is my way to interpret the numpy definition for axes = 0. Take the last 0 axis of x. If the last one axis of x is a vector, then taking the last 0 axis is taking something below/simpler than a vector, which is a scalar. The first 0 axis of y, is to go beyond a vector which is a matrix.

A single instance of x in this case is each scalar value of x, and a single instance of y is the whole y matrix. To unwrap the ‘onion’ we will travel on x by axis 0, then axis 1, or a nested loop, and multiply each instance of x(i,j) by y then sum results. Since in our case x(i,j) is a scalar, no summing is necessary. No axis reduction takes places for axes=0, but the output matrix is transformed. The resulting shape is a combination of all axes of x and y.

(2,3) tensordot (2,3) => (2,3,2,3)x

       [2, 3, 4]])

Tensordot With Parameter Axes = 1

When N = 1, we will take each instance of x (2,3) along last one axis, so that will give us two vectors of length 3, and perform the dot product with each instance of y (2,3) along first axis…

Wait, that doesn’t work because for x we have 2 vectors of length 3, and for y we have 3 vectors of length 2. If we try that the dotproduct on x and y, we will get a ValueError, as shown in the code below.

We can either transpose y to get it in the shape of (3,2), transpose x to get it in the shape of (3,2), or use tensordot tuple notation to specify which axis to use for dotproduct. That is the power of tuple notation, instead of reshaping the tensor, you can simply specify along which axis you want to perform the dot product. Tensordot With Parameter Axes = 2

When axes parameter is two, we take the last two axes of x, with the first two axis of y. To follow the same wording used before, we perform the dot product for each instance of x with each instance of y. For the 2D tensors, we end up taking the whole tensor of x and the whole of tensor of y, multiply element wise, and add all the resulting element together to return a scalar.

(2,3) tensordot (2,3) => scalarnp.tensordot(x,y, axes = 2)same asnp.tensordot(x,y, axes = ((1,0),(1,0)))same asnp.tensordot(x,y.transpose(), axes = ((1,0),(0,1)))same asnp.dot(x.ravel(), y.ravel())Out: 
array(88)

An instance of x is designated by the axes parameter. You can understand it as “each unit of x that will partake in the dotproduct with each unit of y”. For all the examples above, when axes =0, each instance is a single scalar. For axes =1 or ((1),(1)), each instance is a vector. For axes = 2 or ((1,0),(1,0)), each instance is the full 2D tesnor. No matter the dimension of each instance, it get’s reduced to a scalar. Thus another understanding of axes designation is “along which axes to perform the reduction (multiplication/addition)”. We can illustrate reducing the two matrices to a scalar for the case when axes = 2 by flattening the tensors into vectors and performing a dot product.

Multidimensional Tensordot and Tuple Notation

With the dot-product, you can flatten (or unravel) all the dimensions from an instance into a vector, and perform dot product with another vector. This is my own terminology and it only for illustrative purposes.

Each instance, in some sense, is vector from a reshaped tensor. For example when x has a shape of (2,4) and y shape of (4,2), and axes = ((1),(0)), fill vectors a_i with x rows (instead of columns) and fill vector b_j with y columns (instead of rows).

a_0 = x[0,:] 
a_1 = x[1,:]b_0 = y[:,0]
b_1 = y[:,1]
  
out_(0,0) = np.dot(a_0,b_0)
out_(0,1) = np.dot(a_0,b_1)
...
out_(1,1) = np.dot(a_1,b_1)

Another example of unraveling a tensor into its instances, when x has a shape of (2,4) and y shape of (4,2), and axes = ((1,0),(0,1))

a = x[:,1], x[:,0] = [x1, ...x8]
b = y[1,:], y[0,:] = [y1, ...y8]out = np.dot(a,b)

By now, my hope is that the examples above have built some intuition on tuple notation and how each axis designation shapes the resulting tensor.

Review the following notes (stellar handwriting with my mouse :D) to better visualize how tensor axes get reduced in the output tensor during the operations.

The axes parameter doesn’t expect each axis to be in order like some of the examples above where axes = ((2,1,0),(2,1,0)). Given a tensor x of shape (a,b,c) and tensor y of shape (d,e,f), you can perform the tensor dot in any order of axis you want as long as the axis length matches, or the axis can be broadcasted. For example, when we performed a dot product with axes = 0, the scalar value was broadcasted to each vector or tensor. An example of shape matching, if we want to do a tensor dot with the first vector of length 2, the second vector must also be of length 2. Here is how that will look.

x.shape = (a,c,b)y.shape = (f,d,e)np.tensordot(x,y, axes = ((a,b,c)),(d,e,f)))len(a) = len(d)
len(b) = len(e)
len(c) = len(f)

The result is that we have a 3x3 tensor where each value at (i,j) is a single tensordot on x(i,6,2) and y(j,6,2).

Takeaways

Tensordot performs a dot-product (element wise multiplication, followed by addition) along the axes define by the axes parameter.

Axes parameter can take integer inputs to capture some more common cases of tensordot, or tuple inputs that define which axis to perform the dot-product on:

axes = 0:

Each scalar from x multiplied by full y tensor. No axis reduction, the output tensor shape is a combination of x shape and y shape.

axes = 1:

This is a common dot-product common used often in schools and early matrix algebra. The output is reduced by the vector dot-products (‘vector reduction’).

np.tensordot(x,y.transpose(), axes = 1)
np.tensordot(x,y, axes = ((1),(1)))
np.tensordot(x,y.transpose(), axes = ((1),(0)))

axes = 2:

Element wise multiplication/full addition of last two axes of x, with first 2 axes of y. The output is reduced by the matrix dot-product (‘matrix reduction’).

For a 2D tensor, the output will be a scalar. You can extend that to N dimensions, for N dimensional tensors, the output will be a scalar if axes =N.

All of these operations below are equivalent.

np.tensordot(x,y, axes = 2)
np.tensordot(x,y, axes = ((1,0),(1,0)))
np.tensordot(x,y.transpose(), axes = ((1,0),(0,1)))
np.dot(x.ravel(), y.ravel())

axes = tuple:

One way to imagine this is to unravel x and y in the axes order presented by the tuple, then perform vector dot product. The operation reduces the output shape by the axes defined in the tuple, which get reduced to scalar values.
